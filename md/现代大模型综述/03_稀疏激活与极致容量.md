# 第三章：稀疏激活与极致容量

在模型规模不断扩张（Scaling Laws）的指引下，大语言模型的参数量已经突破万亿级别。然而，全连接的稠密网络（Dense Models）面临着一个物理极限：如果每个 Token 在每次前向传播时都要激活网络中的每一个参数，那么所需消耗的 FLOPs（计算量）与计算资源将是不可承受的。

因此，**“稀疏性（Sparsity）”**成为了突破算力墙的核心设计原则。本章将详细介绍当前工业界最主流的混合专家模型（MoE）带来的**条件计算（Conditional Computation）**，以及前沿的 Engram 架构所提出的**条件记忆（Conditional Memory）**补全方案。

## 1. 稀疏性与条件计算：MoE（混合专家模型）

### 1.1 容量与计算的解耦

传统的 Transformer 每一层的 MLP（多层感知机）对所有输入执行相同的变换。混合专家模型（Mixture-of-Experts, MoE）的核心思想是：**“专业的人做专业的事”**。

- 我们将原本巨大的单个 MLP 拆分成多个小型的独立 MLP 网络，也就是“专家（Expert）”。
- 引入一个**路由网络（Router）**，在每次输入时，只挑选少数几个最适合当前 Token 的专家进行计算（例如 Top-2 或 Top-k 路由）。
- **结果**：模型的**总参数量（Total Parameters）**急剧增加（提升了知识容量），但**每次激活的参数量（Activated Parameters）**保持恒定，从而巧妙地解耦了模型容量和计算成本。

### 1.2 路由与负载均衡

MoE 面临的最大工程挑战是负载均衡：如果在训练中，Router 总是偏好将所有的查询发给少数一两个专家，就会导致计算资源倾斜（所谓的“旱的旱死，涝的涝死”），甚至造成网络通信拥堵。为了解决这一问题，现代 MoE 架构通常会引入辅助的负载均衡损失函数（Load Balancing Loss），强制 Router 对各个专家的调配均匀分布；前沿探索中更引入了无门控依赖（Router-free）或更为确定的哈希分配路径。

## 2. 条件记忆范式：超越纯计算的 Engram 架构

尽管 MoE 极大地扩展了模型的推断能力，但大语言模型在本质上处理着两类截然不同的任务：**组合推理（Compositional Reasoning）**和**知识检索（Knowledge Retrieval）**。

- 组合推理往往需要深层、动态的逐层计算（正是深度网络与 MoE 擅长的领域）。
- 而固定知识（例如实体名词、固定短语、死记硬背的法则）在文本中具备强烈的局部和静态特性。

标准 Transformer 缺乏原生的记忆外挂机制，导致模型被迫“使用高昂的注意力与稠密矩阵乘法”临时运算去还原一些非常静态、刻板的数据规则。这不仅浪费了前置层宝贵的计算深度，更是没有将好钢用在刀刃上。

为了对齐底层计算与语言本身的这两种性质，最新的研究提出了**“条件记忆（Conditional Memory）”**架构，并以 **Engram** 作为具体实例，实现了用低廉检索平替动态计算的新思路。

### 2.1 Engram：现代化 $O(1)$ 查找外挂

Engram 模块是一个建立在传统 $N$-gram 模型之上并完全融入现代大语言模型的外部稀疏记忆检索网络。

**工作机制如下：**

1. **记忆哈希检索 (Hash-based Retrieval)**：
   将上下文（前序连续的几个分词，即 N-gram）进行去重、字典降维与压缩之后，利用非常轻量的哈希算法（Hash），将其映射到一个静态庞大的嵌入矩阵（Embedding Table）中的固定索引位置，并以 $O(1)$ 常数时间检索出一条静态特征向量。这无需参与传统的任何浮点矩阵前向推演。
2. **上下文感知门控 (Context-aware Gating)**：
   提取出的静态记忆具有“局限性”，可能遇到多义词或冲突。因此模型会在将其并入主干之前做一次门控：用模型现有的深层动态隐状态（Hidden States）作为查询向量（Query）与提取出的静态记忆做注意力交互（Gating）。一旦环境不符，门控阀门随之关闭，过滤掉噪音。
3. **架构替补（免去重塑冗余）**：
   通过上述手段，Engram 以纯记忆查找的方式将局部词汇规律（如常用的固定实体词）直接“输送”到对应的层。原本用于强行记忆这些局部静态规律的前置注意力网络被彻底解放出来，去专注于更庞大局面的全局、长上下文依赖上。

### 2.2 基础设施感知的高效解耦 (System Efficiency)

Engram 优于传统 MoE 之处在于，它的检索路由是“确定性的”。

- MoE 的 Router 必须实时等本层隐状态计算完成后才能决定选用哪些专家，往往产生不可隐藏的网络和存储存取延迟。
- 而 Engram 基于当前输入的纯文本字符即可以提前解析索引 ID。这使得底层系统可以启用**预取机制（Prefetching）**：跨越 GPU 和主机的 PCIe 或 NVMe，提前在主内存（Host DRAM）把未来需要用到的庞大参数块读取到显存内（隐藏掉通讯延迟），从而完全绕开昂贵的 GPU HBM（高带宽显存）的容量桎梏。

## 3. 稀疏分配定律：计算（Compute）与记忆（Memory）的最佳配比

给定了同样的稀疏参数配额（Inactive Parameters budget），我们应该把空间分给大量的 MoE 专家，还是分给巨大的 Engram 记忆表？
研究揭示了一种优美的**“U型法则（U-shaped Scaling Law）”**。

在严格控制同等的激活参数量（Iso-FLOPs）下：

- **纯 MoE 策略缺陷**：缺少专门保存静态特征的外挂，模型被迫把宝贵的专业参数用在了浅层重塑记忆上，表现次之。
- **纯 Engram 策略缺陷**：缺少运算层面的动态分支，推理和灵活泛化能力下滑。
- **最佳平衡 (Golden Ratio)**：研究发现在将约 $20\% \sim 25\%$ 的闲置参数容量让渡给 Engram 作为静态字典，而保留 $75\% \sim 80\%$ 作为 MoE 分支时，模型在知识库问答、代码、数学以至于通用的逻辑推理评测基准（Benchmarks）上均获得了全面大幅度的攀升（由于替其节省下了早期的特征推导层，相当于模型变相“加深”了网络深度）。

## 4. 本章总结

从稠密矩阵走向极致稀疏，是我们探索大模型物理天花板的重要一环。MoE 利用 **“条件计算”** 横向扩展复杂逻辑的带宽；而 Engram（或其他类型的 KV 记忆外挂机制）开辟了第二条战线，依靠 **“条件存储检索”** 打破了“知识非得由 GPU 费力重演不可”的魔咒。两者相辅相成，指向了以海量内存作支撑、以动态路由于骨架的下一代 AI 基座航向。
